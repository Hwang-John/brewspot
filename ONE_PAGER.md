# BrewSpot 1페이지 요약

## 프로젝트

`BrewSpot`  
내 취향에 맞는 카페를 찾고, 기록하고, 공유하는 지역 기반 커피 커뮤니티 앱

## 왜 만드는가

카페 정보는 많지만, 사용자는 여전히 "내 취향에 맞는 카페"를 찾기 어렵다.  
기존 지도 서비스는 장소 정보에는 강하지만, 커피 취향, 실제 후기, 홈카페 기록, 지역 커뮤니티까지 연결하지는 못한다.

## 해결 방식

1. 지도 기반 카페 탐색
2. 커피 특화 리뷰
3. 취향 기반 추천
4. 지역 커뮤니티
5. 홈바리스타 기록

## 핵심 사용자

1. 20~30대 카페 탐방 사용자
2. 홈바리스타/장비 관심층
3. 지역 기반 추천과 기록을 중시하는 사용자

## MVP 핵심 기능

1. 이메일/Gmail/Kakao/Naver 로그인
2. 위치 기반 지도 탐색
3. 카페 상세 정보
4. 리뷰/별점/사진 업로드
5. 저장 기능
6. 근처 추천
7. 급상승 카페
8. 카페 랭킹
9. 커뮤니티 게시판
10. 홈바리스타 피드
11. 마이페이지
12. 관리자 페이지

## 차별점

1. 단순 장소 검색이 아니라 `커피 취향 중심 구조화 리뷰`
2. 지역 기반 추천과 커뮤니티 결합
3. 홈카페 기록과 카페 탐방 경험을 하나의 앱 안에서 연결

> [AI 추가 제안]
> 경쟁 전략은 `네이버플레이스 대체`가 아니라 `플레이스가 못하는 취향/관계/기록 영역 집중`이 더 적절하다.

## 개발 구조

1. iOS 앱
2. Backend API
3. PostgreSQL
4. 이미지 저장소
5. 관리자 웹
6. SSO 연동
7. 지도/장소 API

## 로그인 정책

1. 이메일, Gmail, Kakao, Naver 로그인 지원
2. SSO 토큰은 서버에서 검증
3. 최소 개인정보만 저장
4. iOS 심사 시 Apple 로그인 필요 여부 별도 확인

## 보안/개인정보 원칙

1. 최소 수집
2. HTTPS 전 구간 적용
3. 토큰 서버 검증
4. 관리자 권한 분리
5. 위치정보와 소셜 로그인 처리방침 명시

## 왜 지금 가능한가

1. 한국 커피 소비 문화는 여전히 강하다
2. 지역 커뮤니티와 하이퍼로컬 사용 습관이 이미 형성돼 있다
3. 커피 기록/취향/커뮤니티를 결합한 버티컬 앱 포지션이 가능하다

## 1차 성공 기준

1. 특정 지역 3~5곳에서 카페 데이터 밀도 확보
2. 리뷰 생성과 저장 행동이 반복되는 구조 확보
3. 추천 클릭률과 재방문율 확인
4. 커뮤니티 초기 활성화
