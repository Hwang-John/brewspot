초기 운영 지역은 아래 3곳으로 고정한다.

2. 연남

이번 1차 시드는 지역별 8개씩 총 24개 카페를 기준으로 사용한다.

## 3. 필수 필드 검수 결과

아래 필드는 24개 카페 모두 채웠다.

1. 이름
2. 주소
3. 위도
4. 경도
5. 카테고리
6. 대표 메뉴명
7. 대표 메뉴 가격
8. 가격대 메모
9. 한 줄 설명
10. 분위기 태그
11. 특징 문장
12. 운영 시간

반영 파일:

1. `CAFE_SEED_TEMPLATE.csv`
2. `SUPABASE_CAFE_SEED.sql`

검수 메모:

1. 영업시간과 메뉴 가격은 공개 매장 정보와 최근 후기 기반으로 정리했다.
2. 좌표는 주소 기반 공개 지오코더로 보강했다.
3. 골목 단위 주소는 실제 지도 앱과 수 미터 차이가 날 수 있으므로 Supabase 실입력 전 한 번 더 눈으로 확인한다.

## 5. 최소 운영 정책 확정

MVP 운영 정책은 아래만 먼저 고정한다.

1. 허위 리뷰 금지
2. 협찬 미표시 광고성 리뷰 금지
3. 욕설, 혐오, 개인정보 노출 금지
4. 신고 접수 후 `유지 / 숨김 / 삭제 / 제한` 중 하나로 처리
5. 반복 위반자는 단계적으로 작성 제한

세부 기준은 `COMMUNITY_POLICY_DRAFT.md`에 최종안으로 반영한다.

## 6. 실입력 전 체크

1. 기존 더미/legacy 데이터가 섞여 있으면 `SUPABASE_RESET_CONTENT.sql` 실행
2. Supabase `cafes` 테이블에 `SUPABASE_CAFE_SEED.sql` 적용
3. `TEST_ACCOUNTS_TEMPLATE.csv` 기준 테스트 계정 15개 생성
4. 테스트 계정 준비 후 `SUPABASE_REVIEW_SEED.sql` 또는 `REVIEW_SEED_TEMPLATE.csv` 적용
5. `SUPABASE_VERIFY.sql`로 카페 24개 / 리뷰 36개 / legacy review 여부 확인
6. 지도 화면에서 좌표와 카테고리 표기가 기대대로 보이는지 확인
