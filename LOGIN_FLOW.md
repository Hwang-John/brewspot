# BrewSpot 로그인/회원가입 플로우

최종 기준일: 2026-04-24

## 1. 목적

이 문서는 BrewSpot의 현재 로그인 구현 범위와, 출시 전 남은 인증 검증 항목을 정리한다.

## 2. 현재 지원 방식

### 현재 구현 완료

1. 이메일 로그인

### 2차 이후 검토

1. Google 로그인
2. Apple 로그인
3. Kakao 로그인
4. Naver 로그인

## 3. 현재 앱 동작 기준

1. 앱 실행 시 세션이 없으면 로그인 화면으로 진입한다.
2. 현재 버전에는 비회원 둘러보기 흐름이 없다.
3. 로그인 완료 후 홈/지도/상세/마이페이지 흐름으로 진입한다.
4. 리뷰 작성과 저장 기능은 로그인 세션이 있는 상태를 전제로 한다.

## 4. 현재 화면 기준

### 로그인 화면

1. 이메일 로그인
2. 이메일 회원가입 진입

### 회원가입 화면

1. 닉네임 입력
2. 이메일 입력
3. 비밀번호 입력
4. 회원가입 완료 후 로그인 화면 복귀

현재 미구현:

1. Google / Apple 버튼
2. Kakao 버튼
3. 비회원 둘러보기
4. 계정 연결 / 해제 UI
5. 프로필 이미지 선택
6. 약관 동의 UI

## 5. 인증 흐름 기준

### 이메일 로그인

1. 이메일 / 비밀번호 입력
2. Supabase Auth 로그인
3. 사용자 프로필 보장 로직 실행
4. 홈 진입

### 이메일 회원가입

1. 닉네임 / 이메일 / 비밀번호 입력
2. Supabase Auth 회원가입
3. 프로필 생성 트리거 또는 보장 로직 반영
4. 이후 로그인

## 6. 계정 연결 정책

문서 기준 정책은 아래와 같다.

1. 현재 MVP는 `email`만 로그인 수단으로 본다.
2. 이메일이 같더라도 자동 병합하지 않는다.
3. 계정 연결은 현재 세션이 있는 상태에서만 허용한다.
4. 대표 로그인 수단이 하나도 남지 않으면 unlink를 허용하지 않는다.

현재 상태:

1. 정책 문서는 있음
2. 실제 계정 연결 UI / 로직은 아직 미구현
3. 소셜 로그인과 계정 연결은 이번 버전에서 제외

## 7. 출시 전 남은 확인 항목

1. 로그인 에러 메시지와 재시도 흐름 점검
2. 이메일 인증 여부에 따른 안내 문구 점검
3. 회원가입 후 `public.users` 반영 흐름 점검
4. 테스트 계정 기준 end-to-end QA 점검

실제 설정 체크는 `AUTH_PROVIDER_SETUP_CHECKLIST.md`를 기준으로 진행한다.

## 8. 보안 체크 포인트

1. 소셜 로그인 재도입 시 OAuth 토큰은 Supabase 또는 서버 검증 흐름을 따른다.
2. 이메일만으로 동일 사용자 자동 병합하지 않는다.
3. 탈퇴 / 연동 해제는 재인증 정책과 함께 설계한다.
4. 소셜 로그인 재도입 시 제공자별 수집 항목은 개인정보처리방침에 명시한다.
