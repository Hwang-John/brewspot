# BrewSpot Next Steps

## 목적

이 문서는 BrewSpot 프로젝트를 실제 제작 단계로 넘기기 위한 다음 작업 우선순위를 정리한 문서다.

---

## 지금 당장 해야 할 일

### 1. GitHub 저장소 정리

- [ ] GitHub 웹에서 기본 브랜치를 `main`으로 변경
- [ ] 변경 후 `master` 브랜치 삭제 여부 결정

설명:

- 지금 `main` 브랜치는 이미 올라가 있음
- 다만 GitHub 기본 브랜치를 바꾸기 전에는 `master` 삭제를 바로 하지 않는 것이 안전함

---

### 2. MVP 최종 고정

- [ ] 최소 MVP를 아래 5개 기능으로 확정
  - 로그인
  - 지도
  - 카페 상세
  - 리뷰
  - 마이페이지
- [ ] 이번 버전에 넣지 않을 기능 확정
  - 커뮤니티 고도화
  - 홈바리스타 고도화
  - 랭킹 고도화
  - 중고장터
  - 커머스

설명:

- 범위를 줄여야 실제로 만들 수 있음
- 비개발자 바이브코딩 환경에서는 기능을 많이 넣는 순간 진행이 멈출 가능성이 큼

---

### 3. 초기 지역 확정

- [ ] 초기 지역 3곳만 선택
- [ ] 각 지역별 카페 후보 10개씩 수집

추천 예시:

1. 성수
2. 연남
3. 망원

설명:

- 지도형 앱은 초기 데이터 밀도가 매우 중요함
- 전국 단위보다 지역 집중이 훨씬 현실적임

---

### 4. 로그인 정책 확정

- [ ] 현재 정책 유지 여부 확인
  - 이메일
  - Gmail
  - Kakao
  - Naver
- [ ] iOS 심사 시 Apple 로그인 필요 여부를 출시 직전에 다시 체크

설명:

- 지금 기획 문서는 네 요청대로 `메일 + Kakao + Gmail + Naver`로 정리됨
- 다만 실제 App Store 심사에선 Apple 로그인 관련 검토가 필요할 수 있음

---

### 5. Supabase 실구축 시작

- [ ] Supabase 프로젝트 신규 생성
- [ ] `SUPABASE_MINI_SCHEMA.sql` 실행
- [ ] 테이블 생성 확인
- [ ] 더미 카페 데이터 입력

우선 넣을 테이블:

1. users
2. user_identities
3. cafes
4. reviews
5. bookmarks

## 내가 놓치기 쉬운 포인트

1. 로그인보다 초기 데이터가 더 중요할 수 있음
2. 추천 알고리즘보다 카페 데이터 밀도가 먼저임
3. 정책 문서는 출시 직전에 급하게 만들면 위험함
4. iOS 심사에서는 로그인/개인정보/신고 기능이 생각보다 중요함
5. 기능을 늘리기보다 완성도를 높이는 쪽이 더 유리함
