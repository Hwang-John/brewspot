# BrewSpot 바이브코딩 개발 순서 제안

## 1. 전제

이 문서는 비개발자가 바이브코딩 방식으로 앱을 만들 때,  
현실적으로 어디까지 먼저 만들고 무엇을 뒤로 미뤄야 하는지 정리한 계획이다.

---

## 2. 핵심 원칙

1. 처음부터 모든 기능을 만들지 않는다.
2. 로그인, 지도, 카페 상세, 리뷰, 마이페이지까지만 먼저 만든다.
3. 백엔드는 최대한 BaaS 기반으로 시작한다.
4. 중고장터와 커머스는 뒤로 미룬다.

## 4. 비개발자에게 현실적인 기술 선택

### 가장 쉬운 조합

1. iOS 앱: SwiftUI 또는 노코드/로우코드 앱 빌더
2. 백엔드: Supabase
3. 관리자: 웹 어드민 템플릿 또는 Supabase 대시보드 일부 활용
4. 파일 저장: Supabase Storage
5. 로그인: Supabase Auth + Apple/Google/Kakao 연동

### 피해야 할 것

1. 처음부터 자체 인증 시스템 만들기
2. 처음부터 자체 추천 엔진 만들기
3. 처음부터 결제/중고거래 정산 기능 넣기
4. 전국 단위 모든 카페를 초반에 다 다루기

---

## 5. 네가 놓치기 쉬운 포인트

1. 카페 데이터는 누가 처음 넣는가
2. 리뷰가 없으면 앱이 비어 보인다는 점
3. 관리자 페이지 없이는 커뮤니티 운영이 어렵다는 점
4. 로그인보다 탈퇴/신고/차단이 나중에 더 중요해질 수 있다는 점
5. 앱스토어 심사에서는 정책 문서와 신고 기능이 생각보다 중요하다는 점
