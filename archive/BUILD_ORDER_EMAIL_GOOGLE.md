# BrewSpot 첫 구현 순서

## 기준

이 문서는 비개발자 바이브코딩 기준으로 `Email + Google` 로그인부터 시작하는 가장 현실적인 구현 순서를 정리한다.

---

## 왜 Email + Google부터 시작하나

1. Supabase 기본 연동이 가장 쉽다.
2. Kakao/Naver보다 초기 디버깅 난도가 낮다.
3. 실제 앱 구조를 먼저 완성하기 좋다.
4. 로그인 이후 Kakao/Naver는 2차 확장으로 붙일 수 있다.

> [AI 추가 제안]
> 기획 문서에는 `메일 + Kakao + Gmail + Naver`를 유지해도 된다. 하지만 실제 첫 개발은 `Email + Google`로 가는 것이 훨씬 성공 확률이 높다.

---

## 1단계: Supabase 기본 연결

- [ ] 프로젝트 생성 완료
- [ ] `SUPABASE_MINI_SCHEMA.sql` 실행
- [ ] `users`, `user_identities`, `cafes`, `reviews`, `bookmarks` 생성 확인
- [ ] 앱에 Supabase URL / Publishable key 연결

완료 기준:

- 앱이 Supabase에 연결된다
- `cafes` 조회 테스트가 된다

---

## 2단계: Email 로그인

- [ ] Supabase Auth에서 Email 활성화
- [ ] 이메일 회원가입 화면 만들기
- [ ] 이메일 로그인 화면 만들기
- [ ] 로그인 성공 후 세션 유지 확인

완료 기준:

- 회원가입 가능
- 로그인 가능
- 로그아웃 가능

---

## 3단계: Google 로그인

- [ ] Supabase Auth에서 Google Provider 설정
- [ ] Google OAuth 정보 등록
- [ ] 앱에서 Google 로그인 버튼 연결
- [ ] 서버/세션 정상 동작 확인

완료 기준:

- Google 로그인 가능
- 기존 이메일 계정과 충돌 없는지 확인

---

## 4단계: 카페 데이터 넣기

- [ ] `cafes` 테이블에 10개 이상 데이터 입력
- [ ] 위치 정보 포함
- [ ] 대표 메뉴 포함
- [ ] 성수/연남/망원 중심으로 데이터 구성

완료 기준:

- 지도 또는 리스트에서 최소 10개 카페가 보임
