# BrewSpot iOS 개발 체크리스트

이 문서는 비개발자도 쉽게 따라갈 수 있도록 지금 당장 해야 할 일을 정리한 체크리스트입니다.

## 1. MVP 핵심 기능 확정
- [ ] 지도 기반 카페 탐색
- [ ] 카페 상세 정보
- [ ] 리뷰 작성 / 별점
- [ ] 근처 카페 추천
- [ ] 요즘 뜨는 카페 / 랭킹
- [ ] 커뮤니티 기본 게시판
- [ ] 로그인 기능

## 2. 로그인 / 계정 정책
- [ ] 이메일 로그인 확정
- [ ] Gmail 로그인 확정
- [ ] Kakao 로그인 확정
- [ ] Naver 로그인 확정
- [ ] iOS 앱이라면 Sign in with Apple 심사 대응 여부 확인
- [ ] 소셜 로그인 연동 + 계정 연결(identity linking) 구조 설계

## 3. 백엔드 / 데이터 구조 결정
- [ ] API 서버 방식 결정
- [ ] PostgreSQL 또는 Supabase 기반 DB 결정
- [ ] 이미지 저장소 방식 결정
- [ ] 필수 테이블/컬렉션 목록 정리
  - users
  - user_identities
  - cafes
  - reviews
  - review_images
  - bookmarks
  - community_posts
  - community_comments

## 4. 위치 / 리뷰 / 카페 데이터 구조 설계
- [ ] 위치 권한 요청 시점 설계
- [ ] 리뷰 항목 구조 확정
  - 별점
  - 사진
  - 코멘트
  - 분위기
  - 작업 적합도
  - 시그니처 메뉴
- [ ] 카페 데이터 필드 확정
  - 이름
  - 주소
  - 위도/경도
  - 대표 메뉴
  - 태그/카테고리

## 5. 초기 서비스 지역 선정
- [ ] 집중 지역 3~5곳 선정
  - 추천: 성수, 연남, 망원, 전포, 행궁동

## 6. 화면 흐름 우선순위
- [ ] 로그인 / 회원가입 화면
- [ ] 홈 지도 화면
- [ ] 카페 상세 화면
- [ ] 리뷰 작성 화면
- [ ] 마이페이지 / 커뮤니티 화면 (2차)

## 7. 개인정보 / 정책 정리
- [ ] 개인정보처리방침 수집 항목 정리
- [ ] 위치 정보 활용 범위 정리
- [ ] 소셜 로그인 수집 항목 정리
- [ ] 탈퇴/데이터 삭제 정책 정리

## 8. 관리자 운영 범위
- [ ] 신고 처리
- [ ] 콘텐츠 관리
- [ ] 사용자 관리
- [ ] 감사 로그 / 운영 정책 기본 설계

## 우선 작업 순서
1. MVP 핵심 기능과 로그인 방식을 확정
2. 백엔드/DB 구조를 간단히 설계
3. 첫 화면 흐름(로그인 → 지도 → 카페 상세 → 리뷰)부터 구현 시작
