# BrewSpot App Store 스크린샷 구성안

최종 기준일: 2026-04-24

## 목적

이 문서는 App Store 제출용 스크린샷을 어떤 화면 순서로 준비할지 정리한다.

## 기본 방향

1. 첫 3장은 기능보다 사용 장면이 바로 보이게 구성
2. 로그인보다 탐색/상세/저장/리뷰 흐름을 먼저 노출
3. 텍스트 오버레이는 짧고 명확하게 유지

## 권장 5장 구성

### 1장 탐색 홈

화면:
홈 화면 + 검색 + 카테고리 + 지도 일부

오버레이 문구 초안:
`지도에서 취향에 맞는 카페를 빠르게 탐색`

### 2장 카페 상세

화면:
카페 상세 상단 + 대표 메뉴 + 태그 + 소개

오버레이 문구 초안:
`대표 메뉴와 분위기를 한눈에 확인`

### 3장 리뷰

화면:
리뷰 목록 또는 리뷰 작성 시트

오버레이 문구 초안:
`방문한 카페를 리뷰로 기록`

### 4장 저장

화면:
상세의 저장 상태 또는 저장한 카페 목록

오버레이 문구 초안:
`마음에 드는 카페를 저장해 다시 보기`

### 5장 마이페이지

화면:
마이페이지 활동 요약 + 최근 활동

오버레이 문구 초안:
`저장과 리뷰를 나만의 취향 기록으로`

## 촬영 체크

1. 실제 Supabase 데이터가 반영된 상태에서 촬영
2. 리뷰 수와 저장 수가 0이 아닌 화면 우선 사용
3. 닉네임과 이메일은 테스트 계정 기준으로 정리
4. 너무 긴 카페명이나 문구는 촬영 전 화면 확인

## 촬영 전 선행 조건

1. 카페 24개 시드 반영
2. 리뷰 36개 시드 반영
3. 테스트 계정 준비
4. 이메일 로그인 동작 확인
5. 상세 / 저장 / 마이페이지 흐름 정상 확인
