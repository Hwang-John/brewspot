# BrewSpot 위치정보 처리 기준 메모

최종 정리일: 2026-04-24

## 1. 현재 구현 판단

현재 BrewSpot iOS 앱은 `위치 사용 중 허용(When In Use)` 권한을 요청할 수 있으며, 아래 범위에서 현재 위치를 활용한다.

현재 구현 범위:

1. `내 위치` 중심으로 지도 이동
2. 현재 위치와 카페 간 거리 표시
3. 현재 위치 기준 가장 가까운 카페 안내
4. 카페 좌표와 현재 위치를 같은 지도 맥락에서 표시

현재 구현 기준 메모:

1. 현재 위치는 앱 실행 중 UI 계산에만 사용한다.
2. 현재 구현에는 현재 위치를 Supabase 등 서버로 저장하거나 전송하는 로직이 없다.
3. 백그라운드 위치 추적, 위치 기록 히스토리 저장, 위치 기반 알림은 없다.

## 2. 향후 위치기능 도입 시 선행 조건

아래 기능 중 하나라도 추가하면 위치정보 관련 문서를 다시 공개 기준으로 전환한다.

1. 현재 위치 중심 지도 이동
2. 근처 카페 자동 추천
3. 거리순 정렬을 홈 핵심 정렬로 승격
4. 이용자 위치를 서버 저장 또는 로그 적재
5. 이용자 위치 기반 알림 또는 기록 기능

## 3. 향후 반영 체크리스트

위 기능을 도입할 경우 출시 전에 아래를 함께 반영해야 한다.

1. `Info.plist` 위치 권한 문구 추가
2. 앱 내 위치 동의 및 거부 흐름 추가
3. 개인정보처리방침 위치정보 항목 개정
4. App Privacy 답변 재검토
5. 위치정보 관련 고지 문구 및 이용약관 공개 여부 재검토
6. 위치정보법상 신고 또는 추가 의무 해당 여부 재검토

## 4. 참고 메모

현재 판단은 `앱 내 현재 위치 접근은 있지만 서버 저장/전송은 없는` 현행 코드 기준이다.  
정식 출시 전에는 개인정보처리방침, App Privacy, 위치기반서비스 약관 공개 필요 여부를 함께 다시 확인한다.
